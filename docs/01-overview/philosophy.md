# Philosophy

Opo is built on a clear set of design principles that define its identity and goals. The "Opposite of Python" approach is central to its design.

## The "Opposite of Python" Principle

While Python is designed to be highly readable through natural language-like keywords and significant whitespace, Opo takes a more symbolic and explicit approach.

1.  **No Significant Whitespace**: Opo uses explicit block delimiters (e.g., `[]`) for better structure and clarity, especially in large codebases.
2.  **Statically Typed**: Types are declared and checked at compile-time, reducing runtime errors and improving performance.
3.  **Explicit Over Implicit**: There are no implicit type conversions. If you want to convert an integer to a float, you must do it explicitly.
4.  **Keyword-Light, Symbolic-Heavy**: Opo uses unique symbolic operators (e.g., `!!` for print, `? :` for if-else, `@` for loops) to keep the language minimalist and expressive.

## Core Values

### Precision
Opo rewards precision. Every variable has a type, every function has a signature, and every operation is clear. This results in less ambiguity and more reliable software.

### Efficiency
A language should not stand in the way of performance. Opo's runtime is optimized for speed, and its memory model is designed to be efficient.

### Simplicity
A language with a small grammar is a language that is easy to learn and master. Opo's syntax is consistent and predictable, making it a powerful tool in the hands of a skilled developer.
